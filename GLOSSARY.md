# Glossary 📖

Shared terminology used across all tracks in this repository.

| Term | Definition |
|---|---|
| **Algorithm** | A step-by-step procedure for solving a problem or performing a computation. |
| **API** | Application Programming Interface — a contract defining how software components communicate. |
| **Array** | An ordered, indexed collection of elements (called "list" in Python, "slice" in Go). |
| **Assertion** | A statement that checks if a condition is true; used in exercises for self-checking. |
| **Backpropagation** | The algorithm for computing gradients in neural networks by propagating error backward. |
| **Boolean** | A data type with exactly two values: true and false. |
| **Callback** | A function passed as an argument to another function, to be called later. |
| **Class** | A blueprint for creating objects that bundles data (attributes) and behavior (methods). |
| **Closure** | A function that captures and remembers variables from its enclosing scope. |
| **Compiler** | A tool that translates source code into machine code before execution (Go, Rust, TypeScript). |
| **Concurrency** | Running multiple tasks that can make progress independently (not necessarily simultaneously). |
| **CNN** | Convolutional Neural Network — a deep learning architecture for spatial data like images. |
| **Data Structure** | A way of organizing and storing data for efficient access and modification. |
| **Decorator** | A function that wraps another function to extend its behavior (Python, TypeScript). |
| **Dictionary** | A key-value mapping data structure (called "map" in Go, "object" in JS, "HashMap" in Rust). |
| **Dynamic Typing** | Type checking at runtime rather than compile time (Python, JavaScript). |
| **Embedding** | A dense vector representation of data (words, images) in a continuous space. |
| **Exception** | An error condition that disrupts normal program flow (called "panic" in Go/Rust). |
| **Float** | A number with a decimal point, stored using IEEE 754 floating-point representation. |
| **Function** | A reusable block of code that takes inputs (parameters) and optionally returns a value. |
| **Generator** | A function that yields values one at a time, enabling lazy evaluation (Python). |
| **Generic** | A way to write code that works with multiple types without sacrificing type safety. |
| **Gradient Descent** | An optimization algorithm that iteratively adjusts parameters to minimize a loss function. |
| **Hash Map** | A data structure mapping keys to values with O(1) average lookup time. |
| **Immutable** | A value that cannot be changed after creation (strings in Python, tuples, Rust defaults). |
| **Integer** | A whole number without decimals. |
| **Interface** | A contract specifying which methods a type must implement (explicit in TS/Go, implicit in Go). |
| **Interpreter** | A tool that executes source code line-by-line at runtime (Python, JavaScript). |
| **Iterator** | An object that produces a sequence of values one at a time. |
| **JSON** | JavaScript Object Notation — a text format for structured data exchange. |
| **Lambda** | An anonymous (unnamed) function, typically short and inline. |
| **Lifetime** | Rust's compile-time mechanism for ensuring references are valid for the correct duration. |
| **LLM** | Large Language Model — a neural network trained on large text corpora to generate/understand text. |
| **Loop** | A control structure that repeats a block of code until a condition is met. |
| **Method** | A function attached to an object or type. |
| **MLOps** | Machine Learning Operations — practices for deploying and maintaining ML models in production. |
| **Module** | A file or package that organizes related code for reuse and encapsulation. |
| **Mutable** | A value that can be changed in place after creation (lists in Python, variables in Rust with `mut`). |
| **Neural Network** | A computing system inspired by biological neurons, composed of layers of connected nodes. |
| **None/Null/Nil** | A special value representing "no value" or "nothing" (`None` in Python, `null` in JS, `nil` in Go). |
| **OOP** | Object-Oriented Programming — organizing code around objects that combine state and behavior. |
| **Ownership** | Rust's system for memory management — each value has exactly one owner at a time. |
| **Pointer** | A variable that stores the memory address of another variable (Go, Rust, C). |
| **Promise** | A JavaScript object representing the eventual result of an asynchronous operation. |
| **RAG** | Retrieval-Augmented Generation — enhancing LLM responses with external knowledge retrieval. |
| **Recursion** | A function that calls itself to solve smaller instances of the same problem. |
| **Reference** | An alias that allows access to a value without taking ownership (Rust's borrowing). |
| **RNN** | Recurrent Neural Network — a deep learning architecture for sequential data. |
| **Scope** | The region of code where a variable is accessible. |
| **Set** | An unordered collection of unique elements. |
| **Static Typing** | Type checking at compile time rather than runtime (TypeScript, Go, Rust). |
| **String** | A sequence of characters representing text. |
| **Struct** | A composite data type grouping related fields (Go, Rust — similar to class without methods). |
| **Tensor** | A multi-dimensional array — the core data structure in deep learning frameworks. |
| **Trait** | Rust's mechanism for defining shared behavior (similar to interfaces). |
| **Transformer** | A neural network architecture based on self-attention, powering modern LLMs. |
| **Tuple** | An ordered, immutable collection of a fixed number of elements. |
| **Type** | A classification of data that determines what operations are valid on a value. |
| **Variable** | A named storage location for a value in a program. |

> This glossary covers terms used across the repository. For language-specific terms, see each track's README.
